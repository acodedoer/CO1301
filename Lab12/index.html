1. Introduction

  1. In this lab, you will be using dot product to calculate whether one object (player) is within the field of view of another object (drone).

  2. This worksheet introduces some new concepts. However, take it steady and you will have no problems with it.

  3. The idea of this exercise is that we want to allow the player to sneak up behind the drone, and that is possible because the drone cannot see the player if the player is behind it.

3. Calculating and Displaying the Dot Product

The steps that need to be completed to find out whether the drone can see the player are:

  • Calculate the vector from the drone to the player.
  • Find out the facing vector of the drone.
  • Calculate the dot product between these two vectors.
  • Perform check:
    • If the dot product is greater than 0 then the angle between the two vectors is less than 90 degrees, if the dot product is less than 0 then the angle is greater than 90 degrees, and if the dot product is 0 then the angle is exactly 90 degrees.
    • Therefore if the dot product is less than 0 then the player is behind the drone and not visible to the drone.
The tasks that follow describe the implementation of these steps.

  1. Calculate the vector from the drone to the player by subtracting the x,y and z components of the drone's position from that of the player's position. This should be implemented in the game-loop, and the x, y and z components of the resulting vector should be stored in a vector (created from the Vector3 struct you created in 2.6) called drone2PlayerVector.

  2. To calculate the drone's facing vector, you have to carry out some work with the drone's matrix. But We haven't covered matrices yet, so let us use a less efficient approach. Since the facing vector is just the vector that describes the direction in which an object is facing, we can find out what that vector is if we move the drone along its local z-axis by any value, and calculate the vector between the old and the new position of the drone (of course we will then have to move the drone back). Let's put this "trick" in a function that takes in a Vector3 and a model, calculates and assigns the components of the model's facing vector to the components of the Vector3 it received as parameters. 

    void calculateFacingVector( IModel *model, Vector3 &facingVector)
{
    float initialPosX = model->GetX();
    float initialPosY = model->GetY();
    float initialPosZ = model->GetZ();

    model->MoveLocalZ(1.0f);

    float finalPosX = model->GetX();
    float finalPosY = model->GetY();
    float finalPosZ = model->GetZ();

    facingVector.x = finalPosX - initialPosX;
    facingVector.y = finalPosY - initialPosY;
    facingVector.z = finalPosZ - initialPosZ;

    model->MoveLocalZ(-1.0f);
}

    Add the function to your code so you can call it to calculate the drone's facing vector.

  3. Assuming that the model of the drone is called "drone" and we have a Vector3 variable ready to store the facing vector of the guard: 

    Vector3 droneFacingVector;

    You would then call the function like this:

    calculateFacingVector( drone, droneFacingVector );

    Note that the function does not return any value, but can update the values of droneFacingVector.x, droneFacingVector.y and droneFacingVector.z as they are passed by reference.

  4. Now write a function that will calculate the dot product of two vectors v and w. The function should take in as parameters two Vector3 vectors V and W, and return their dot product (a float). The function declaration (or function prototype) looks like this: 

    float calculateDotProduct( Vector3 v, Vector3 w );
    Add this before your program's main function.

  5. The function prototype specifies that you need to supply the function with two vectors. The body of the function needs to calculate the dot product of the two vectors. The dot product is calculated using the following formula: v·w = (v.x * w.x + v.y * w.y + v.z * w.z)

    Below the closing brace of the main function, define the calculateDotProduct function. You need to:

    • write the function declaration without the ending semi-colon
    • add curly brackets (braces) and place the body of the code within the curly brackets.
    float calculateDotProduct( Vector3 v, Vector3 w )
{
    // body of code here
}

  6. Now pass droneFacingVector and drone2PlayerVector as parameters to the calculateDotProduct function to find the dot product of the two vectors. Assign the result to a float variable named dotProduct. 

    float dotProduct = calculateDotProduct( droneFacingVector, drone2PlayerVector );

  7. Finally, apply the following test:

    • If dotProduct > 0 then the angle is less than 90 degrees, if dotProduct < 0 then the angle is greater than 90 degrees, and if dotProduct == 0 then the angle is exactly 90 degrees.
    • Therefore if dotProduct < 0 then the player is behind the drone and not visible to the drone.

    Display the dot-product and visibility of the player on the screen:

    • At the top left of your screen, display the dot product calculated at every frame in the format: "Dot-Product: 0".
    • At the top right corner of your screen, display the visibility status of the player. If the dot product is less than 0, display "Visibility: Hidden" else display "Visibility: Visible".

  8. Move your player around and note how the dot product and the visibility status change.
