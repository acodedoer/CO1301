Lab 12 - Dot Product

1. Introduction

  1. In this lab, you will be using dot product to to calculate whether one object (player) is within the field of view of another object (drone).

  2. This worksheet introduces a number of new concepts. However, take it steady and you will have no problems with it.

  3. The idea of this exercise is that we want to allow the player to sneak up behind the drone. The drone cannot see the player if the player is behind it.

2. Setting Up

  1. Create a new TL-Engine project called "Lab12_DotProduct_Project".

  2. A set of media files are available on BlackBoard. You need to use the Floor.x mesh to create a floor model, drone.x to create a drone model, and either sierra.x, Casual_A.x,marine.x or the Bikergirl.x to create a player model.

  3. Create models for the floor, the player and the drone. You will need to scale the drone down by a factor of 0.05 (use drone->Scale(0.05f)) and make sure it is positioned above the floor so that it appears to be hovering.

  4. Set up the W, A, S and D keys to move the player, e.g. 

    if( myEngine->KeyHeld( Key_W ) )
{
    player->MoveLocalZ( kPlayerSpeed );
}

  5. You will need to calculate the vector from the drone to the player each frame. In order to do this, take the location of the player and subtract the location of the drone from it. This should be implemented in the game-loop, and the x, y and z components of the vector should be stored in appropriately named float variables e.g. drone2PlayerVx, drone2PlayerVy and drone2PlayerVz.

  6. We also need the drone's facing vector. In order to calculate this you have to carry out some work with its matrix. We haven't covered matrices yet, so a function that will calculate the facing vector from a given model is provided below: 

    void facingVector( IModel *model, float &x, float &y, float &z)
{
    float matrix[16];

    model->GetMatrix( matrix );

    x = matrix[8];
    y = matrix[9];
    z = matrix[10];
}

  7. Assuming that the model of the drone is called "drone" and we have three floating point variables ready to store the facing vector of the guard: 

    float facingVx;
float facingVy;
float facingVz;

    You would then call the function like this:

    facingVector( drone, facingVx, facingVy, facingVz );

    Note that the function does not return any value, but is able to update the values of facingVx, facingVy and facingVz as they are passed by reference.

3. Calculating and Displaying the Dot Product

  1. Now write a function that will calculate the dot product of two V and W, it will be used to calculate the dot product if the two vectors derived in the last section. The function should take in as parameters the componenets of V and W, and return the dot product (a float). The function declaration (or function prototype) looks like this: 

    float dotProduct( float Vx, float Vy, float Yz, float Wx, float Wy, float Wz );
    Add this before your program's main function.

  2. Below the closing brace of the main function, write the actual function. You need to:

    • write the function declaration without the ending semi-colon
    • add curly brackets (braces) and place the body of the code within the curly brackets.
    float dotProduct( float Vx, float Vy, float Yz, float Wx, float Wy, float Wz )
{
    // body of code here
}

  3. The function prototype specifies that you need to supply the function with two vectors. The body of the function needs to calculate the dot product of the two vectors. The dot product is calculated using the following formula:

    v·w = (vxwx + vywy + vzwz)

  4. To recap: in order to find out whether the drone can see the player you:

    • Use the facingVector() function to find out the facing vector of the drone.
    • Calculate the vector from the drone to the player.
    • Calculate the dot product between these two vectors.

  5. Having done these steps you finally apply the following test:

    • Assuming the facing vector (the orientation) of the drone be v and the vector from the drone to the player be w.
    • If v·w > 0 then the angle is less that 90 degrees, if v·w < 0 then the angle is greater that 90 degrees, and if v·w = 0 then the angle is exactly 90 degrees.
    • Therefore if v·w < 0 then the player is behind the guard and not visible to the guard.

  6. Now, dsiplay the dot-product and visibility of the player on the screen:

    • At the top left of your screen, display the the dot product calculated at every frame in the format: "Dot-Product: 0".
    • At the top right corner of your screen, display the visibility status of the player. If the the dot product is less than or equal to 0, display "Visibility: Hidden" else if the dot product is greater than 0 display "Visibility: Visible".

  7. Move your player around and note how the dot product and the visibility status change.
