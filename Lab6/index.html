  6. A vector is a way of describing movement or direction. In 3D a vector is simply three numbers: x, y and z. However, these numbers now represent movement or direction instead of a location. Vectors are used throughout games development. The vector of movement for your particle is ( 0, -0.005, 0 ).

  7. Vectors are written using round brackets. The values within the round brackets are separated from one by commas. The first number is the x value. The second number is the y value. The third value is the z value.

  8. A vector of movement ( 0, -0.005, 0 ) means that the particle is advancing by:

    • a value of 0 along the x axis
    • a value of -0.005 along the y axis
    • a value of 0 along the z axis

  9. Declare three floating point variables: vectorX, vectorY and vectorZ. Set the value of vectorX to 0, the value of vectorY to 0.005 and the value of vectorZ to 0. The variables should be created outside the game loop.

  10. Change the MoveY() command in your code so that you are now moving the particle model by vectorY.

  11. Add a MoveX() command so that you are moving the particle model by vectorX.

  12. Add a MoveZ() command so that you are moving the particle model by vectorZ.

  13. Compile and run your program. Your particle should be dropping down the screen as before.

  18. As you can see, one of the immediate advantages of using a vector is that you don't have to change any code within the game loop. All you have to consider is the direction of movement and set up the appropriate vector. This is how all movement within a graphics engine is dealt with. All models within 3D graphics are moved using vectors.
